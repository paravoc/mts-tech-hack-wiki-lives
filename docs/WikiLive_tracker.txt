WikiLive: трекер доработок и сдачи
Дата: 10.04.2026

Назначение документа:
Рабочий трекер для отслеживания, что уже готово, что критично доделать и что нужно сдать по 6 контрольным точкам.

1. Текущее состояние проекта

Сильные стороны проекта сейчас:
- Есть рабочий backend на C++23.
- Есть интеграция с MWS Tables.
- Есть CRUD страниц и хранение страниц в WikiPages.
- Есть рендер живых вставок из MWS.
- Есть web UI, автопредпросмотр и демо-данные.
- Есть AI-помощник для suggest-insert.
- Есть тесты и локальный воспроизводимый запуск.

Главный риск проекта сейчас:
- Не закрыт обязательный editor-контур по условиям трека.
- Нет полноценной вставки живой таблицы в страницу как блока.
- Нет backlinks.
- Нет slash-menu с горячими клавишами.
- Нет локального кеша браузера и сценария восстановления после перезагрузки.
- Нет проверяемой совместной работы.
- Нет Docker / docker-compose с однострочным запуском.

2. Что критично доделать

Критично для оценки жюри:
[ ] Вставка существующей таблицы MWS Tables в тело страницы как живого блока.
[ ] Slash-menu с поддержкой горячих клавиш.
[ ] Backlinks между страницами.
[ ] Local cache браузера и восстановление содержимого после перезагрузки.
[ ] Совместная работа в документе в проверяемом виде.
[ ] Docker / docker-compose и одна команда запуска.

Важно, но после обязательного контура:
[ ] Полировка UX и приближение к Design Kit.
[ ] Визуальный редактор вставок: ячейка, строка, колонка, таблица.
[ ] AI-подсказки как усиление, а не ядро продукта.
[ ] Комментарии, версии, граф связей как дополнительные фичи.

3. Что лучше убрать из фокуса

Чтобы не потерять баллы, сейчас не стоит ставить в центр работы:
- обучение собственной модели;
- сложный локальный AI pipeline;
- отдельную desktop-версию;
- сложные бонусные фичи раньше обязательных;
- глубокую полировку AI до закрытия обязательных сценариев.

4. Контрольная точка 1: обязательный функциональный контур

Статус: частично готово.

Уже есть:
- страницы;
- живые вставки отдельных значений;
- автопредпросмотр;
- мягкое автосохранение на уровне UI/backend.

Нужно добить:
[ ] Живую таблицу как блок.
[ ] Slash-menu /table, /page, /callout, /heading.
[ ] Горячие клавиши минимум для основных блоков.
[ ] Backlinks и ссылки между страницами.
[ ] Local cache браузера.
[ ] Совместную работу.
